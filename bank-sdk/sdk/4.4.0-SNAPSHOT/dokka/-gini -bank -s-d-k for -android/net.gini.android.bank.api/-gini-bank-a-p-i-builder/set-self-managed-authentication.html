setSelfManagedAuthentication

public GiniBankAPIBuilder setSelfManagedAuthentication(Boolean enabled)

Enable self-managed authentication: the SDK will not authenticate API requests and no SessionManager (or client credentials) are required.

When enabled, your GiniHttpClientProvider's OkHttpClient is responsible for adding the Authorization header to API requests (for example with your own application or network interceptor - either works, because the SDK installs no authentication of its own in this mode). Your access token is never passed through the SDK.

A custom GiniHttpClientProvider must be set via setHttpClientProvider, otherwise building will throw an IllegalStateException. Disabled by default.

Return

The builder instance to enable chaining

Parameters

enabled

pass true to authenticate API requests yourself
